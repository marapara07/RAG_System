# RAG System - Internal Knowledge Assistant

## Project Overview

This project represents the current implementation of an enterprise-oriented Retrieval-Augmented Generation (RAG) system designed for internal organizational knowledge assistance.

The application allows employees and administrators to authenticate, access internal documentation, upload and reindex documents, ask natural language questions, and receive context-aware answers generated by a locally deployed Large Language Model.

The current implementation focuses on multilingual document-based question answering, secure user authentication, scalable architecture, and local AI inference.

---

## Current Features

The current version includes:

- User authentication
- Role-based access (Employee / Administrator)
- Document upload functionality
- Full document reindexing
- PDF and TXT document processing
- Chroma vector database integration
- PostgreSQL relational database integration
- FastAPI backend services
- React frontend interface
- Local LLM inference through Ollama
- Multilingual question answering
- Source attribution for generated answers

## Backend Setup

Open a terminal in the project root.

Activate the virtual environment:

```bash
.venv\Scripts\activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Start the backend:

```bash
uvicorn backend.main:app --reload
```

The backend runs at:

```text
http://127.0.0.1:8000
```

---

## Frontend Setup

Open another terminal.

Navigate to frontend:

```bash
cd frontend
```

Install dependencies:

```bash
npm install
```

Start the frontend:

```bash
npm run dev
```

The frontend runs at:

```text
http://localhost:5173
```

## RAG Pipeline

The application currently follows the following workflow:

### 1. Document Loading

TXT and PDF documents are loaded from:

```text
backend/storage/documents
```

### 2. Text Extraction

The content is extracted and converted into plain text.

### 3. Chunking

Documents are split into smaller semantic chunks.

### 4. Embedding Generation

Chunks are transformed into embeddings using:

```text
nomic-embed-text
```

### 5. Vector Storage

Embeddings are stored in ChromaDB.

### 6. Retrieval

Relevant chunks are retrieved based on the user question.

### 7. Generation

The retrieved context is passed to the local language model.

### 8. Answer Generation

The model generates a multilingual answer based only on the retrieved context.

## Current Limitations

The current version still has several limitations:

- Full reindexing is required after adding documents
- Large document collections may increase indexing time
- Retrieval quality depends on chunk quality
- Incremental indexing is not yet implemented
- Deployment is currently local only
